Customer Churn Prediction

Predicting which customers are likely to churn using feature engineering and ensemble machine learning techniques. The project aims to help businesses identify at-risk users and suggest targeted retention actions.

Project Overview

This project builds a churn prediction pipeline that ingests historical customer data (usage patterns, transactions, demographics), engineers predictive features, and trains supervised models to estimate churn probability. Outputs include model evaluation metrics, a list of top predictors, and suggested retention actions for high-risk segments.

Typical Pipeline

  1. Data loading & exploratory data analysis (EDA)
  2. Data cleaning and missing value handling
  3. Feature engineering (aggregations, behavioral features, categorical encoding)
  4. Train/test split, cross-validation and class balancing (SMOTE or weighting)
  5. Model training (Logistic Regression, Random Forest, XGBoost), hyperparameter tuning
  6. Evaluation (accuracy, precision, recall, F1-score, ROC-AUC) and feature importance

How to run (quick)

  1. Clone the repo: 
    git clone https://github.com/Jaikishan1228/Customer_churn_prediction_SI_04.git
  2. Install dependencies: 
    pip install -r requirements.txt
  3. Open the notebook: 
    jupyter notebook Customer_Churn_Prediction.ipynb
  4. Run the cells for data preparation, modeling and evaluation. Check the `models/` and `outputs/` folders for trained models and result artifacts.
